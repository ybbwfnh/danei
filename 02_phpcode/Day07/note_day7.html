[return[值] } 2)函数调用 [$变量] = 函数名称(参数) 3)原理:函数不调用,不执行.定义函数时,会将函数放到内存代码段中.当调用函数时去内存中函数名称所在 位置中执行函数体,执行完将控制权移交回给调用函数的位置上. 4).函数命名规范 a. 各种地方不能出现中文. b. 函数名称不区分大小写 c. 尽量以动词开头,例如 getinfo,setinfo. 5) 参数 a. 参数:程序员跟函数沟通的媒介. b. 参数个数至少0个. c. 参数类型可以是任何类型 d. 可选参数带[],必选不带[]...自定义函数中,没值的叫必选参数,有值叫可选参数 必选参数一定在可选参数前面. function($n,$m=1){ $n 必选,$m 不写就默认是1. ... } 6)返回值 a. 返回值:函数执行的结果. b. 返回值的个数只能是0个或者1个 c. 返回值可以是任何数据类型 d. 返回值和参数没有必然关系. e. 结束函数执行的功能,并且把执行权归还给调用函数的位置上. 4. 变量的作用域 1)局部变量 :变量在代码段中(函数)中有效. a.动态局部变量 b.静态局部变量. static :作用在函数中,不释放内存,一般实现计数. 2)全局变量 在函数外面的是全局变量,在代码段中,必须声明 a.在代码段中,声明全局变量 在函数中 增加global = $n; $n = 12; 方法2是 $GLOBALS['n']=12 ; 就是12了 方法3引用传递 在变量中增加&地址符 $n = 7; funciton test(&$n){ $n +=5; return $n; } test($n); echo $n; //就等于12了,而不是7; 3)超全局变量 :变量在任何地方都有效,且不用声明 所有的预定义变量都是超级变量 5.预定义变量 1)$_POST :接受表单 POST 传输方式 以名/值对的形式接受 2)$_GET :表单 GET传输方式,以名/值对的形式接受数据. 3)$GLOBALS :声明全局变量 4)$_FILES :接受上传文件数据信息. 5)$_SESSION :会话机制 变量的跨页面传输 6)$_COOKIE :会话机制 变量的跨页面传输 7)$_REQUEST :等同于 $_POST $_GET $_COOKIE的接受. 8)$_SERVER :服务器信息 9)$_ENV :环境变量 说明: a.预定义变量都是以数组形式存储的. 例如: $user = $_POST['user']; 接受表单中的一个元素 print_r($_POST) 看到表单的所有元素. b.所有的预定义变量都是超级变量 6. 函数函数库 1)func_num_args: 获得函数的参数个数.一共有多少参数 2)func_get_args: 获得所有参数值组成的数组. 3)func_get_arg: 获得参数组成的数组中的某一个参数值 4)function_exists: 判断函数名是否存在 BOOL结果 7.变量函数 原理:将函数名称赋给变量. 使用我们的变量时带小括号使用,相当于函数调用
